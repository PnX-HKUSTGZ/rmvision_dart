## 串口协议变更检查表

- [ ] 明确变更原因和协议版本。
- [ ] 同步更新视觉端与电控端字段顺序、类型和单位。
- [ ] 检查 packed 结构体大小。
- [ ] 检查包头和 CRC 覆盖范围。
- [ ] 定义新增字段的默认值、无效值和边界值。
- [ ] 使用原始字节流完成收发验证。
- [ ] 测试断包、错包、CRC 错误和旧版本数据。
- [ ] 更新 [Send.msg](../../src/auto_aim/auto_aim_interfaces/msg/Send.msg)、[packet.hpp](../../src/rm_serial_driver/include/packet.hpp)及知识库接口说明。
- [ ] 提供兼容策略或同步升级步骤。
- [ ] 提供回滚版本。

## 双相机与目标切换验收表

- [ ] base/outpost 相机设备与物理位置一致。
- [ ] 两路相机内参和外参没有混用。
- [ ] 两路图像、`Send_pnp` 和 `Send_fused` 均持续发布。
- [ ] `target_id=1` 选择 base，`target_id=0` 选择 outpost。
- [ ] 非当前目标相机不会污染最终 `/Send`。
- [ ] 切换后旧消息在超时内失效。
- [ ] 单相机模式和双相机模式分别验证。
- [ ] 相机断开、Topic 超时和重新连接行为明确。
