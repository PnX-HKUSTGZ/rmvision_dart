# 飞镖视觉 Q&A 问题快查

使用方式：先按“快速检查”缩小模块，再进入[部署实践手册](03_deployment_dart.md)执行完整操作。已经确认根因并具有复用价值的问题，应补充到[调试记录](07_debug_records.md)。

## 1. 相机和图像

### Q1：相机没有图像，或者双相机只有一路

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 相机是否被系统识别；序列号是否与角色对应；`camera_start_mode`；图像 Topic 是否存在；是否有其他进程占用设备 |
| 常见原因 | 设备标识错误、USB/供电问题、Launch 只启用单相机、驱动异常、设备被旧进程占用 |
| 处理 | 停止重复进程；核对 [launch_params.yaml](../../src/vision_bringup/rm_vision_bringup/config/launch_params.yaml)；单独验证相机驱动；重新插拔并确认供电 |
| 验证 | base/outpost 图像和 `camera_info` 均持续发布，角色与实际安装位置一致 |

### Q2：画面正常，但二值图全黑

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 镜头档位、曝光、增益、相机标定文件、HSV/绿色优势阈值 |
| 常见原因 | 更换镜头或现场亮度后继续使用旧参数；绿灯通道值未达到阈值；曝光过低 |
| 处理 | 先固定曝光和增益；查看原图通道值；逐项调整二值化，不同时修改几何筛选 |
| 验证 | 绿灯在不同帧均保留，背景白字和反光没有大面积进入二值图 |

### Q3：白字、反光或高亮边缘被识别为绿灯

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 候选区域的绿色优势、圆度、填充率、宽高比和发光面积 |
| 常见原因 | 只使用亮度或单一绿色阈值；ROI 过宽；形态学将碎片连接成近似圆形 |
| 处理 | 优先增强颜色区分，再收紧几何规则；使用同一段负样本验证，不凭单帧调参 |
| 验证 | 负样本不再进入最终候选，同时不同距离的真实绿灯仍能通过 |

## 2. 检测、PnP 与稳定性

### Q4：检测框存在，但 PnP 距离明显错误

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 标定文件、图像分辨率、镜头、`pnp_circle_radius_mm`、检测圆半径 |
| 常见原因 | 内参与实际镜头不匹配；绿灯物理半径配置错误；检测到的不是完整发光圆 |
| 处理 | 用已知距离样本核对内参和物理尺寸；不要用融合阈值掩盖 PnP 基础错误 |
| 验证 | 多个距离下误差趋势合理，且更换目标距离后不会按固定比例偏差 |

### Q5：角度或 `stability` 在 0/1 间频繁跳变

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 原始检测是否丢帧；目标角变化；卡尔曼参数；融合有效率；最终稳定逻辑 |
| 常见原因 | 检测阈值位于临界值；滤波过敏；点云有效与无效交替；目标切换后状态未重置 |
| 处理 | 先稳定检测，再检查角度滤波，最后检查融合和输出稳定逻辑；不要直接强制稳定作为正式修复 |
| 验证 | 固定目标下切换次数显著减少，移动目标仍能及时响应 |

### Q6：移开绿灯后仍发送上一帧有效结果

| 项目 | 内容 |
| --- | --- |
| 快速检查 | `Send_pnp`、`Send_fused`、最终 `/Send` 的时间戳和字段；`send_mux` 超时行为 |
| 常见原因 | 无目标分支未覆盖全部字段；融合或 mux 缓存了旧消息；数据源超时未显式失效 |
| 处理 | 沿三个阶段逐级比较，确认在哪一级没有恢复无效状态；检查超时和目标切换重置 |
| 验证 | 目标消失后在规定超时内进入未知/无效状态，不继续发送旧距离和角度 |

## 3. Livox、TF 与融合

### Q7：有原始点云，但没有 `/livox/accum_points`

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 累积节点是否运行；目标 frame；TF 是否可查询；点云时间戳；距离裁剪范围 |
| 常见原因 | TF 缺失或重复；frame 名错误；时间戳下无变换；所有点被距离过滤 |
| 处理 | 先检查 TF 树和原始点云 frame，再查看累积节点日志；临时可视化确认裁剪结果 |
| 验证 | 累积点云持续更新，frame 正确，点数与场景规模相符 |

### Q8：点云存在，但融合距离无效

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 图像 ROI；外参；`roi_scale`；有效距离范围；PnP 距离门限；`min_points` 和 MAD |
| 常见原因 | 投影偏移、ROI 内点数不足、背景点占主导、有效距离配置与实际场景不符 |
| 处理 | 先看点云投影视频；修正外参后再调 ROI；最后调整稳健测距阈值 |
| 验证 | ROI 中目标点稳定，融合距离在重复样本中有效且不被近处结构抢占 |

### Q9：投影点云整体向一个方向偏移

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 相机角色、frame 名、平移和旋转符号、光学坐标轴、标定文件 |
| 常见原因 | base/outpost 外参混用；轴方向错误；相机光学 frame 与普通 link frame 混淆 |
| 处理 | 使用多个已知位置验证外参；先修坐标系和符号，不通过无限放大 ROI 补偿 |
| 验证 | 不同距离和方向下投影误差均可接受，而非只对单个样本正确 |

### Q10：融合结果经常回退到 PnP

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 融合有效标志、点数、MAD、有效范围、PnP 距离门限和回退配置 |
| 常见原因 | ROI 无效；雷达距离被范围过滤；回退区间与当前赛场距离不匹配 |
| 处理 | 统计回退发生条件；先修复点云和外参；只在明确需要时调整回退策略 |
| 验证 | 正常雷达样本优先使用融合结果，雷达异常时能稳定退化且状态语义明确 |

## 4. 舱门状态

### Q11：绿灯被遮挡时一直输出未知

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 原始点云 Topic、舱门判断开关、ROI、远处开门证据、最少点数和超时 |
| 常见原因 | 舱门 ROI 没有覆盖开口；点云超时；远处证据不足；轴方向错误 |
| 处理 | 使用原始点云检查中央 ROI；分别录制开门、遮挡和未知样本；按证据调参 |
| 验证 | 开门遮挡稳定输出 `light_detected=2`，无可靠证据时仍保持未知而非误判 |

### Q12：舱门未完全打开却被判断为已打开

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 中央 ROI 是否吃到门框；近点和远点数量；距离上下限；连续确认帧数 |
| 常见原因 | ROI 过大；固定结构提供了错误远点；近点阈值不足；单帧噪声直接改变状态 |
| 处理 | 缩小到实际开口区域；分开近点与远点证据；保留连续帧确认 |
| 验证 | 门框、滑台和偶发噪声不会触发开门，真实开门能在可接受时间内确认 |

## 5. 双相机与目标选择

### Q13：`target_id` 改变后仍输出另一相机结果

| 项目 | 内容 |
| --- | --- |
| 快速检查 | `/target_id` 实际值；base/outpost Topic；检测门控；`send_mux` 参数和超时 |
| 常见原因 | 目标编号理解反了；Topic 重映射错误；旧缓存未超时；相机角色接反 |
| 处理 | 分别 echo 两路融合结果和最终 `/Send`；确认 `1=base`、`0=outpost`；检查 mux 来源 |
| 验证 | 切换后最终输出来自正确命名空间，旧目标不会继续刷新 |

### Q14：只改了 `node_params.yaml`，双相机参数没有生效

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 对应参数是否在 Launch 中按角色覆盖 |
| 常见原因 | 把 `node_params.yaml` 的默认兜底值误认为最终值 |
| 处理 | 检查 [vision_bringup.launch.py](../../src/vision_bringup/rm_vision_bringup/launch/vision_bringup.launch.py)，硬件关系和双相机半径优先改 `launch_params.yaml` |
| 验证 | 使用 `ros2 param get` 查看节点运行时最终参数 |

## 6. YAML 与配置

### Q15：融合节点启动即退出，日志显示轴参数不合法

| 项目 | 内容 |
| --- | --- |
| 快速检查 | `door_forward_axis`、`door_lateral_axis`、`door_vertical_axis` 的 YAML 类型和值 |
| 常见原因 | 未加引号的 `y` 等值被 YAML 解析为布尔值，或正负号丢失 |
| 处理 | 使用带引号的显式字符串，例如 `"+x"`、`"+y"`、`"+z"` |
| 验证 | 节点成功启动并在日志中输出预期轴映射 |

## 7. 串口与扫码枪

### Q16：串口节点运行，但电控收不到有效数据

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 设备名、权限、波特率、校验位、停止位、包头、结构体大小和 CRC |
| 常见原因 | 设备编号变化；协议双方字段不同步；packed 结构或类型长度不一致 |
| 处理 | 对照 [packet.hpp](../../src/rm_serial_driver/include/packet.hpp)逐字节核对；抓取原始包；确认电控协议版本 |
| 验证 | 连续收发通过，所有字段在边界值和无目标状态下均正确 |

### Q17：扫码枪没有生效，或 serial 模式仍启动扫码节点

| 项目 | 内容 |
| --- | --- |
| 快速检查 | `dart_input_mode`、设备名、扫码字符串、正则、编号与偏置范围、Launch 条件 |
| 常见原因 | 模式配置错误；字符串格式不匹配；槽位未填满；旧 Launch 无条件启动节点 |
| 处理 | 先单独验证扫码枪节点输出，再检查检测节点订阅；确认模式为 `serial` 或 `barcode` |
| 验证 | `/barcode/scan_profile` 内容正确，serial 模式下扫码节点不参与链路 |

## 8. 延迟、内录和导出

### Q18：开启 debug 后延迟明显升高

| 项目 | 内容 |
| --- | --- |
| 快速检查 | `debug`、点云绘制、调试图订阅者、CPU/IO、`/latency` |
| 常见原因 | 每帧图像绘制、编码和发布；大量点云投影；RViz 或录像占用资源 |
| 处理 | 比赛关闭 debug；减少点云绘制；使用低优先级内录；赛后离线生成可视化 |
| 验证 | `/latency` 恢复稳定，串口输出不再因资源争用抖动 |

### Q19：没有生成 rosbag

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 是否通过 `dart.sh` 启动；`enable_rosbag_recorder`；保存路径权限；启动延迟；磁盘空间 |
| 常见原因 | 直接运行 Launch；内录开关关闭；目录不可写；后台进程启动失败 |
| 处理 | 查看 `dart.sh` 输出和后台进程；使用临时环境变量验证；检查保存目录 |
| 验证 | 生成新 bag，metadata 和数据库持续更新，停止后可被 `ros2 bag info` 读取 |

### Q20：断电后 bag 无法读取

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 数据库文件是否存在；metadata 是否缺失或不完整 |
| 常见原因 | 断电时 sqlite3 尚未完成索引和 metadata 写入 |
| 处理 | 保留原始目录，执行 `ros2 bag reindex <bag_dir>`；再执行 `ros2 bag info` |
| 验证 | bag 可列出 Topic 和时间范围，导出工具能够读取 |

### Q21：导出视频损坏、缺帧或时间轴不一致

| 项目 | 内容 |
| --- | --- |
| 快速检查 | bag 是否完整；输出帧率；导出是否中断；图像与 `Send` 时间戳范围 |
| 常见原因 | 未先 reindex；进程异常退出导致视频容器未收尾；帧率配置不匹配 |
| 处理 | 删除不完整的导出目录后重新导出；按实际相机帧率设置参数；保留原始 bag |
| 验证 | 视频可正常播放，时长与 bag 接近，标注和图像事件对齐 |

## 9. Livox Viewer 找不到雷达

| 项目 | 内容 |
| --- | --- |
| 快速检查 | 是否已有 Livox 驱动或系统自启动程序占用设备 |
| 常见原因 | `dart.service` 或手动启动的视觉程序已经连接雷达 |
| 处理 | 先确认占用进程；在安全条件下停止视觉服务后再使用 Viewer |
| 验证 | Viewer 能连接；重新启动视觉后 `/livox/lidar` 恢复 |
